## Authority and Override Rules

APPLIES WHEN: Any reply using athlete history, memory, prior plan, or inferred context; any case with updated constraints, conflicting signals, or corrected facts

## Rules
- Current athlete report over prior assumptions
- Explicit constraints over inferred preferences
- Safety signals over ambition
- Present readiness over prior plan continuity
- If current report conflicts with memory, retire the conflicting memory on that dimension
- If athlete updates a constraint, re-evaluate from zero on that dimension
- If multiple facts changed, rebuild from current state instead of patching the old plan
- A prior plan is reusable only if current state still supports it; if readiness is unclear, hold or reduce

## Avoid
- Preserving old constraints after the athlete corrects them
- Preserving old optimism after the athlete reports setback signals
- Forcing continuity with a prior plan when current state no longer supports it
- Treating memory as equal to current report or stacking old caveats onto new reality

## Red Flags
- Reply references limitations the athlete just removed
- Reply follows an old plan despite new fatigue, pain, illness, or schedule disruption
- Recommendation sounds driven by continuity rather than current readiness
