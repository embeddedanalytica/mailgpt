## Common Running Prescription Errors

APPLIES WHEN: Any running recommendation involving load, workout choice, comeback, or progression — use as a final check before prescribing

## Rules
- Current run tolerance over target race logic
- Session cost over session name
- Repeatable easy running over isolated ambitious sessions
- Mechanical simplicity: flat, controlled, low-variance running first
- The cheapest next useful step
- If tolerance is uncertain, keep the run easy and predictable
- Add only one cost at a time: frequency or duration or intensity
- Before quality, require stable easy running and normal next-day response
- First quality exposure is small, controlled, and isolated
- If athlete absorbs a week well, repeat once before a bigger jump when risk is recent
- If response is mixed, remove complexity before removing volume

## Avoid
- Prescribing workouts just because the athlete is behind
- Hiding intensity inside "steady," "moderate," "progression," or "strides"
- Pairing long run growth with workout return
- Treating one smooth run as proof
- Using hills, terrain, or pace goals early in a fragile phase
- Letting race timeline override current tolerance

## Red Flags
- "Let's test it"
- "You handled one run well"
- "We'll make up fitness"
- Workout added after inconsistent week
- Long run stretched after recent pain, illness, or travel
- Pace prescription despite unstable energy or symptoms
- Computing a pace target from a goal description (e.g., deriving per-mile pace from "sub-3:30") instead of reading the stored pace from memory — goal descriptions are not pace values; the stored training_baseline has the actual target
- Writing a number that does not appear in the input — weekly mileage, session yardage, rep counts, and durations must be read from memory facts or plan data, not computed or approximated
- Implausible magnitudes: weekly run mileage in the hundreds or thousands, swim volumes above 30,000 yards, stride durations above 40 seconds, effort above 100% — any of these signals a fabricated number
